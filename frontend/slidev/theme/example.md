# Detailed Assessment Breakdown

**Weekly Quizzes (15%)**

- 10 questions per module
- Multiple choice + coding snippets
- Unlimited attempts, best score counts

**Hands-On Projects (25%)**

- 5 major projects aligned with curriculum
- Rubric: Code quality (40%), functionality (40%), documentation (20%)
- Submitted via GitHub, reviewed within 1 week

**Module Exams (45%)**

- 3 comprehensive exams (15% each)
- Mix of theory and practical coding
- 2-hour time limit, closed-book

**Capstone Project (15%)**

- End-to-end ML solution for real business problem
- Team-based (3-4 participants)
- Includes: model development, deployment, presentation

# Identified Risks & Mitigation Strategies

| Risk                          | Probability | Impact | Mitigation Strategy                                                  |
| ----------------------------- | ----------- | ------ | -------------------------------------------------------------------- |
| **Participant Dropouts**      | Medium      | High   | Flexible schedules, recorded sessions, peer support groups           |
| **Technical Issues**          | High        | Medium | Backup cloud accounts, local VMs, 24/7 tech support                  |
| **Content Obsolescence**      | Medium      | Medium | Quarterly curriculum review, modular design, industry advisory board |
| **Insufficient Resources**    | Low         | High   | Pre-provisioned cloud credits, alternative platforms (Kaggle, Colab) |
| **Instructor Unavailability** | Low         | Medium | Backup instructors, guest lecturers, pre-recorded modules            |
| **Budget Overruns**           | Medium      | High   | Contingency fund (20%), phased resource allocation                   |

<div class="mt-4 glass-card p-4">
<strong>Contingency Plan</strong>: All course materials available offline, alternative delivery modes ready (async, self-paced)
