# Product Requirements Document: Presentation Mode for SmartSlate Polaris

## Executive Summary
Build a world-class, industry-standard presentation mode for SmartSlate Polaris learning blueprints. This feature transforms blueprint data into an interactive, animated, full-screen presentation experience with professional-grade UI/UX, brand-compliant design, and comprehensive presentation controls including speaker notes, laser pointer, and presentation settings.

## Project Metadata
- **Project Name**: SmartSlate Polaris Presentation Mode
- **Priority**: High
- **Timeline**: 5-6 weeks
- **Tech Stack**: Next.js 15, TypeScript 5.7, Framer Motion, Recharts, Swiper, Tailwind CSS v4
- **Integration Points**: Blueprint View Page, Blueprint Share Page

## Problem Statement
Users need a professional way to present their learning blueprints to stakeholders, teams, and learners. Currently, blueprints are only viewable as scrollable documents. A presentation mode enables users to:
- Present blueprints in meetings and training sessions
- Share via presentation links
- Export blueprints as presentation-ready PDFs
- Engage audiences with animated, visual content
- Control presentations with professional tools (laser pointer, speaker notes, timer)

## Success Criteria
1. **Presentation Quality**: Industry-standard presentation experience matching tools like Google Slides, Keynote
2. **Brand Compliance**: All UI elements use SmartSlate Polaris glassmorphic design with teal accent colors
3. **Performance**: Smooth 60fps animations, <200ms slide transitions
4. **Accessibility**: WCAG AA compliant with keyboard navigation and screen reader support
5. **Integration**: Seamless integration with existing blueprint view and share pages
6. **Feature Completeness**: Full presentation controls including speaker notes, laser pointer, timer, settings

## User Stories

### Primary User Stories
1. **As a learning designer**, I want to present my blueprint in full-screen mode so that I can engage my audience during presentations
2. **As a presenter**, I want speaker notes visible only to me so that I can reference talking points without showing them to my audience
3. **As a presenter**, I want a laser pointer tool so that I can highlight specific content on slides
4. **As a user**, I want keyboard shortcuts so that I can control the presentation efficiently
5. **As a user**, I want to share presentation links so that others can view my blueprint as a presentation
6. **As a user**, I want to export presentations as PDFs so that I can distribute static versions

### Secondary User Stories
7. **As a presenter**, I want presentation settings (auto-advance, transition speed) so that I can customize the presentation experience
8. **As a user**, I want touch/swipe gestures so that I can present from tablets and touch devices
9. **As a user**, I want animated infographics and charts so that data visualizations are engaging
10. **As a presenter**, I want a timer and slide counter so that I can track presentation progress

## Design Specifications

### Color Palette
- **Primary Teal**: #14b8a6 (brand accent)
- **Background**: Dark gradient (#0f172a в†’ #1e293b в†’ #0f172a)
- **Glass**: rgba(255, 255, 255, 0.1) with backdrop-blur-xl
- **Text**: white (#ffffff) with opacity variants (100%, 80%, 60%, 40%)
- **Success**: #22c55e
- **Warning**: #f59e0b
- **Error**: #ef4444

### Typography Scale
- **Hero (h1)**: 4rem - 8rem (clamp based on viewport)
- **Title (h2)**: 3rem - 5rem
- **Subtitle (h3)**: 2rem - 3rem
- **Body**: 1rem - 1.5rem
- **Caption**: 0.875rem - 1rem
- **Font Family**: System sans-serif stack (from Tailwind)

### Spacing System (Material Design 3 inspired)
- **4px** (0.25rem): Minimal spacing
- **8px** (0.5rem): Small gaps
- **12px** (0.75rem): Default gaps
- **16px** (1rem): Medium spacing
- **24px** (1.5rem): Large spacing
- **32px** (2rem): Section spacing
- **48px** (3rem): Major section spacing

### Animation Guidelines
- **Duration**: 300ms (fast), 600ms (standard), 800ms (slow)
- **Easing**: cubic-bezier(0.6, -0.05, 0.01, 0.99) for entries
- **Stagger**: 100ms delay between elements
- **GPU Properties Only**: transform, opacity (avoid left, top, width, height)
- **Respect**: prefers-reduced-motion media query

### Glassmorphic Effect
```css
.glass {
  background: rgba(255, 255, 255, 0.1);
  backdrop-filter: blur(20px);
  border: 1px solid rgba(255, 255, 255, 0.2);
  box-shadow: 0 8px 32px 0 rgba(0, 0, 0, 0.37);
}
```

## Risks & Mitigation

### Risk 1: Performance on Low-End Devices
- **Mitigation**: Lazy load slides, GPU acceleration, reduce animations on low FPS
- **Fallback**: Static transitions if performance <30fps

### Risk 2: Browser Fullscreen API Inconsistencies
- **Mitigation**: Polyfill for older browsers, graceful degradation
- **Fallback**: Maximized window mode if fullscreen unavailable

### Risk 3: Complex Blueprint Data Parsing
- **Mitigation**: Comprehensive parser tests, error boundaries
- **Fallback**: Show error slide with helpful message

### Risk 4: Large Bundle Size Impact
- **Mitigation**: Code splitting, dynamic imports for layouts
- **Target**: <100KB added to bundle
